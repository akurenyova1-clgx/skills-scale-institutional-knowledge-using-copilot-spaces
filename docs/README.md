# OctoAcme Project Management Docs

Welcome to the central documentation for OctoAcme's project management processes. This README provides a summary of our approach and quick links to the full set of process documents.

## Summary of Project Management Processes

OctoAcme follows a customer-first, iterative, and data-driven delivery model with clearly defined roles (Project Manager, Product Owner, Engineering, QA) and structured phases from initiation through retrospective:

- **Lifecycle**: Initiation → Planning → Execution & Tracking → Release & Deployment → Retrospective & Continuous Improvement
- **Key artifacts**: Project one-pagers, roadmaps, backlogs, risk registers, release checklists, and retrospective notes
- **Roles**: Defined responsibilities for PMs, product owners, developers, QA, and stakeholders
- **Communication cadence**: Regular standups, sprint reviews, stakeholder updates, and async written communication
- **Consistency**: Templates and checklists ensure repeatable, high-quality execution across projects
